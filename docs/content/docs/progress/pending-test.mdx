# 待测试

- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。

- RunningHub 配置：从通用“配置与用户偏好”中独立出来，新增独立管理入口；API Key 加密存入后端 RunningHub 配置域，工作流列表通过独立接口保存，画布 RunningHub 节点改为读取独立配置。
- RunningHub 工作流：通过 curl 导入参数时可识别 `fieldData` 枚举数组，字符串参数会保存 `enumOptions` 并在画布 RunningHub 节点中以下拉框选择。

- 后端安全与部署修复：启动时要求配置 JWT 和加密主密钥，WebDAV 代理改为登录后可用并限制内网目标，文件存储会校验文件 ID，Docker 构建恢复前后端锁文件并统一使用 3040 端口。
- 后台 boss-worker 任务调度日志：任务创建、排队、运行、进度、完成、失败、取消等关键阶段会输出带任务上下文的标准日志，仍需人工跑一轮任务确认日志齐全且无敏感字段。
- 前端后端化修复：登录态恢复会先验证并刷新 token，再连接 WebSocket 和拉取配置；我的素材登录后自动从后端加载；新建、导入和打开画布在后端失败时会提示或返回画布列表。
- 画布新建：默认新建画布改为按现有标题选择未占用的“无限画布 N”，避免已存在“无限画布 2”时再次创建同名画布。
- 画布生图失败处理：生成图片节点会保存实际提交提示词，失败详情保留后端真实错误，失败后重试不再因提示词为空而中断；后端生图任务会把任务模型和提示词补入上游请求，避免 `params` 未带模型时被兼容服务误路由。

- 后端可观测性：新增 `X-Trace-Id` 响应头、标准化 `k=v` 日志、任务与审计 trace 持久化，以及 WebSocket 任务 trace 透传，仍需人工做一轮端到端验证。
